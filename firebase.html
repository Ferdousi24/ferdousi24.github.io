Squirrel Sightings Tracker Using Firebase and Python

Project Overview

This project involves the development of a lightweight Python application that collects and uploads wildlife observation data — specifically squirrel sightings — to a cloud-based Firebase Realtime Database. Each observation includes species name, fur color, and geolocation (latitude and longitude). The project simulates a real-world use case where wildlife data is collected in the field and automatically stored in a centralized, cloud-accessible platform.

The core objective was to explore how cloud databases can be used in geospatial workflows to manage spatial and attribute data efficiently. Firebase was used for its real-time data syncing capabilities, while the Firebase Admin SDK in Python enabled secure, authenticated data transactions.

Key Contributions

  • Database Setup: Configured Firebase Realtime Database to accept structured wildlife observation data.
  • Authentication: Implemented secure access using service account credentials via Firebase Admin SDK.
  • Data Structuring: Designed a consistent schema using JSON format to store species details, colors, and geolocation points.
  • Scripting: Developed Python code to automate the process of generating and uploading squirrel sighting records.

What I Learned

  • Firebase Fundamentals: Gained hands-on experience with setting up and managing a cloud NoSQL database, and connecting it to Python-based applications.
  • Secure API Integration: Learned how to authenticate scripts and securely manage sensitive credentials for cloud services.
  • Practical Geospatial Workflows: Understood how wildlife or environmental data can be structured and streamed from the field to the cloud for further GIS analysis or visualization.
